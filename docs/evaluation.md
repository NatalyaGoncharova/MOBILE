# Evaluation Rubric

Основное задание: проверить сценарий покупки номера до шага ввода ИИН.

Максимум за базовое задание: 100 баллов.

Extra-задание: дополнительно проверить раздел покупки номера из авторизованной зоны.

## Основной тест: покупка номера без авторизации - 100

Тест: `test_unauthorized_user_can_reach_iin_step_from_number_purchase`

- 10: сценарий стартует из неавторизованного состояния приложения;
- 10: открытие flow "Стать клиентом" реализовано через `BecomeClientPage`;
- 8: выбор города реализован через page object, без прямых локаторов в тесте;
- 10: переход в раздел покупки номера реализован через `BecomeClientPage`;
- 12: `NumberPurchasePage` проверяет, что каталог номеров открыт;
- 15: выбор доступного номера реализован в `NumberPurchasePage`, выбранный номер возвращается в тест;
- 10: переход к оформлению заказа реализован через page object;
- 10: `RegistrationPage` проверяет, что открыт экран оформления для выбранного номера;
- 10: тест доходит до шага ввода ИИН и проверяет видимость поля ИИН;
- 5: тест читается как пользовательский сценарий, без низкоуровневых деталей Appium.

## Extra: покупка номера в авторизованной зоне - до 30 бонусных баллов

Extra-тест можно назвать, например:

`test_authorized_user_can_open_number_purchase_catalog`

или, если сценарий должен доходить дальше:

`test_authorized_user_can_start_number_purchase_flow`

- 6: тест выполняет авторизацию через `LoginPage`;
- 6: вход в раздел покупки номера из авторизованной зоны реализован через отдельный метод page object;
- 6: проверяется, что открыт каталог доступных номеров;
- 6: тест выбирает доступный номер или проверяет доступность выбора номера;
- 6: тест проверяет корректный следующий экран или понятное ограничение, если авторизованному пользователю нельзя продолжить покупку.

Extra-задание не должно ломать базовую оценку. Если extra-тест нестабилен или зависит от состояния аккаунта, это нужно явно описать в `README.md` или комментарии к тесту.

## Общие требования к реализации

- page objects описывают действия и проверки на языке продукта;
- локаторы хранятся в `locators.py`;
- в тестах нет `AppiumBy`, `WebDriverWait`, `driver.find_element`;
- используются явные ожидания из `BasePage`;
- `time.sleep` не используется;
- тестовые данные и секреты берутся из `.env`;
- реальные персональные данные не попадают в репозиторий;
- методы, которые вызываются тестами, не содержат `raise NotImplementedError`;
- проект запускается по инструкции из `README.md`.

## Красные флаги

Эти проблемы могут сильно снизить оценку:

- основной тест оставлен с `pytest.skip`;
- сценарий написан одним длинным тестом с локаторами внутри;
- page object содержит `raise NotImplementedError` в методе, который вызывается тестом;
- тесты завязаны на `time.sleep`;
- Page Object содержит проверки, не относящиеся к своему экрану;
- сценарий покупки номера пытается завершить реальную покупку вместо остановки на вводе ИИН;
- extra-сценарий требует специфичный аккаунт, но это нигде не описано;
- используются реальные персональные данные в репозитории;
- APK или секреты закоммичены в Git.
